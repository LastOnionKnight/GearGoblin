### Added
- **`Theme/TtChrome.cs`** (new module) ‚Äî Track 2 chrome helper, the first piece of the ember/frost-blue visual language that replaces TlfTheme over the v0.6.7 ‚Üí v1.0 migration arc. Lifts the palette tokens verbatim from `RUNTIME_PORT.md ¬ß1.3` (the TLF HUD v0.1 design handoff):
  - **Ink** ‚Äî `#14161C` (90% alpha) backdrop, slightly warmer than TlfTheme's `InkPanel` (`#0E1232`). Drops the navy cast.
  - **Ember** ‚Äî `#D67B3C` warm-orange accent. Replaces `Lantern` gold for Track 2. Used for eyebrow labels and severity-critical badges.
  - **Frost** ‚Äî `#F2F2F5` body text + `rgba(178, 178, 178, *)` frame outlines at three opacity stops (1.0, 0.45, 0.22). The doubled-frame chrome uses the 1.0 outer and 0.22 inner stops.
  - **Severity ramp** ‚Äî `SeverityCritical` ember `#D67B3C`, `SeverityWarning` yellow `#E2C36A`, `SeverityNote` frost-blue `#7FC9EE`. Plus `HpGreen #6FBF6A` for match-success and `Farm #FF9D4A` for mismatch.
- **`TtChrome.BeginCard` / `EndCard`** ‚Äî the signature doubled-frame chrome. Outer 2px frost-outline at 0 inset; inner 1px hairline at 6px inset. Implemented via `ImDrawList.ChannelsSplit(2)` so the frame draws on the background channel *behind* content rather than overlaying text. Card backdrop fill in `InkSurface`.
- **`TtChrome.Eyebrow(fonts, label)` / `Quip(fonts, text)` / `Number(fonts, text, color)` / `Pill(text, color)`** ‚Äî parallel API to TlfTheme's composite helpers. All accept the `FontAtlasManager` and gracefully fall back to default font when individual Track 2 font handles are null (which they will be until Brian drops the .ttf files into `Assets/Fonts/`).
- **Five Track 2 font handles** in `FontAtlasManager`:
  - `CormorantBody` ‚Äî Cormorant Garamond Regular @ 15px (body quip prose)
  - `CormorantItalic` ‚Äî Cormorant Garamond Italic @ 15px (Stab. voice, italic emphasis)
  - `JetBrainsMonoBody` ‚Äî JetBrains Mono Regular @ 11px (numerics, iLvl, deltas)
  - `JetBrainsMonoMeta` ‚Äî JetBrains Mono Bold @ 10px (meta labels, severity counts)
  - `EorzeaRune` ‚Äî Eorzea.ttf @ 13px (ornamental rune accent, decorative only)
  
  All optional via the existing `TryBuild` pattern ‚Äî null fallback if the .ttf is missing. **No callers reference them yet** ‚Äî that wiring is v0.6.7.1 work. v0.6.7 ships the loader plumbing only so Brian can drop the fonts in at his own pace without breaking the build.

## [0.6.6.5] ‚Äî Gear table polish + section eyebrow glyph fix

### Changed
- **Equipped Gear table polished** per the v0.6.6.0 `DrawGearTable` TODO block:
  - **Striped rows** ‚Äî alternating `InkPanel` / `InkPanelAlt` backgrounds via per-row `TableSetBgColor(RowBg0)`. Removes `ImGuiTableFlags.RowBg`'s default light-gray stripe which clashed with the dark TlfTheme navy.
  - **Slot column** ‚Äî Press Start 2P 10px in `GoldDim`, uppercased (e.g. `MAINHAND`, `RINGLEFT`).
  - **Item column** ‚Äî Cinzel SemiBold 16px in `Frost`. HQ items get a trailing `‚òÖ` in `GoldBright` rendered inline via `SameLine(0, 6)`.
  - **iLvl column** ‚Äî default ImGui font in `Knife` (steel), right-aligned via `SetCursorPosX(currentX + availW - textWidth)` rather than a hardcoded offset.
  - **Materia column** ‚Äî each meld in `Frost`, with a faint `¬∑` separator in `FrostFaint` between melds for visual hierarchy. Fixed an edge case in the original where `m.SlotIndex` was used as the "last meld" predicate (worked when slots were 0,1,2,3 but unsafe if a slot were skipped); now uses the loop index against the list count.
- **Header row removed** from the gear table ‚Äî the section head supplies context; the column headers were visual noise on the dark TlfTheme surface.
- **`DrawGearTable` signature** now takes `Plugin plugin` as its first parameter (needed for `plugin.Fonts.Pixel` / `plugin.Fonts.CinzelEmphasis` access). Matches the pattern already established for `DrawHero`, `DrawStatsStrip`, and `DrawMateriaAdvisor`. The only call site (`Draw()`) updated accordingly.

### Fixed
- **Section eyebrow glyph rendering as `?`** in the About and Quick Start tabs (`? TLF GEAR DIVISION ¬∑ OPERATIONS BRIEF`, `? TLF MANIFESTO`, etc.). Root cause: `TlfTheme.GlyphSection` was `"‚óá"` (U+25C7 WHITE DIAMOND), which is not in Dalamud's default font glyph range. The two filled-shape glyphs (`‚ñ∂` U+25B6, `‚óÜ` U+25C6) work; the outlined variant (`‚óá`) does not. Swapped to `¬ª` (U+00BB, RIGHT-POINTING DOUBLE ANGLE QUOTATION MARK), which lives in Latin-1 Supplement and is universally available. Affects all eight `TlfTheme.Eyebrow(...)` call sites in `MainWindow.cs` ‚Äî seven on the Quick Start tab (`TLF MANIFESTO`, `WHAT THIS PLUGIN DOES`, `THE LOOP`, `SLASH COMMANDS`, `WHAT YOU'LL SEE IN THE CHARACTER WINDOW`, `WHEN SOMETHING LOOKS WRONG`, `TIPS`) and one on the About tab (`TLF GEAR DIVISION ¬∑ OPERATIONS BRIEF`).
